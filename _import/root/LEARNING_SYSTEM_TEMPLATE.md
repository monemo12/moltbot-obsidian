# 學習系統建置模板 (Learning System Builder Template)

> 版本：v2.0  
> 用途：為任何領域建立個人化、理論驅動的學習系統

---

## 📋 階段一：需求診斷 (Needs Assessment)

### 1.1 學習目標 (Learning Objectives)
**為什麼？** → 決定深度和評估方式（Bloom's Taxonomy）

選擇主要層次（可複選）：
- [ ] 記憶 - 記住事實、術語
- [ ] 理解 - 解釋原理、舉例
- [ ] 應用 - 實際情境使用
- [ ] 分析 - 拆解問題、比較
- [ ] 評鑑 - 批判思考
- [ ] 創造 - 整合設計

---

### 1.2 時間與節奏 (Time & Rhythm)
**為什麼？** → 決定間隔策略和內容切分

- **每日可投入：** _____ 分鐘
- **持續期：** 短期（<3月）/ 長期（>6月）
- **節奏：** 密集式 / 分散式（✅ 更有效）

---

### 1.3 學習偏好 (Preferences)
**為什麼？** → 匹配資源類型（Dual Coding）

- [ ] 閱讀文字
- [ ] 觀看影片
- [ ] 聆聽音訊
- [ ] 做練習/測驗
- [ ] 討論/教學
- [ ] 實作專案

**自主性：** 高（自己安排）/ 低（需要提醒）

---

### 1.4 當前程度 (Current Level)
**為什麼？** → 決定起點（ZPD）

- 完全新手 / 初學者 / 進階者 / 專家

---

### 1.5 執行細節 (Execution Details)
**為什麼？** → 確保系統可落地執行

- **學習範圍：** 明確子領域或主題（避免太廣）
- **成功定義：** 學到什麼程度算達標？
- **資源語言：** 中文 / 英文 / 雙語
- **資源模式：** AI 生成 / 策展現有 / 混合
- **SRS 工具：** Anki / Markdown / 其他
- **提醒需求：** 是/否，頻率
- **進度檢查：** 主動/被動，頻率
- **困難處理：** 換方法/跳過/降級，誰觸發
- **評估標準：** 單元完成的判定條件

---

## 🧠 階段二：學習理論選擇 (Theoretical Framework)

根據診斷結果選擇適用方法：

| 理論 | 用途 | 工具 |
|------|------|------|
| **間隔重複** (SRS) | 長期記憶事實/概念 | SRS 卡片 |
| **提取練習** | 強化記憶、檢測盲點 | 自我測驗 |
| **刻意練習** | 技能提升、針對弱點 | 目標練習+回饋 |
| **精緻化** | 理解複雜概念 | 自我解釋、類比 |
| **交錯學習** | 辨別相似概念 | 混合式練習 |
| **變異練習** | 遷移到不同情境 | 多樣案例 |
| **後設認知** | 監控學習過程 | 學習日誌 |

**產出：** 選定 3-5 個核心理論組合

---

## 🏗️ 階段三：系統組件設計 (System Components)

### 3.1 學習路徑 (Learning Path)
```
領域 → 模組（3-5個）→ 單元 → 檢核點
```
**原則：** 先備知識在前、由簡入繁、每模組 3-5 單元

---

### 3.2 每日執行計畫 (Daily Routine)
**範例（15 分鐘）：**
- 3 分 → SRS 複習
- 8 分 → 新內容學習
- 2 分 → 新增卡片
- 2 分 → 學習日誌

**可調整：** 根據時間和節奏調整比例

---

### 3.3 SRS 卡片系統
**卡片類型：**
- **概念卡** - 定義、原理
- **應用卡** - 情境判斷
- **辨別卡** - 相似概念比較

**原則：** 一卡一概念、用自己的話、加實例

---

### 3.4 進度追蹤
**檔案：** `learning-systems/<領域>-progress.md`

**記錄：** 完成單元 / SRS 統計 / 學習日誌 / 里程碑 / 調整記錄

**檢核：** 每週小測 / 每模組大測

---

## 🔄 階段四：維護與優化 (Maintenance)

### 定期回顧
- **每週：** 檢視進度、識別困難、調整節奏
- **每月：** 後設認知評估、策略調整

### 動態調整信號
- **加速：** SRS >90% 連續一週 → 增加新內容
- **減速：** SRS <70% 持續 → 減少新內容、增加複習
- **轉向：** 資源效果不佳 → 調整資源組合

---

## 👥 角色分工

**學習者：** 提供診斷答案、執行計畫、回饋調整需求  
**系統建置者：** 設計路徑、準備卡片、推薦資源、監控調整、定期提醒

---

## 📦 產出清單

1. `<領域>-system.md` - 學習系統（診斷+理論+路徑+計畫）
2. `<領域>-cards.md` - SRS 卡片庫
3. `<領域>-progress.md` - 進度追蹤
4. `<領域>-resources.md` - 資源清單
